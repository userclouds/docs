Tokens are created using the CreateToken and LookupToken APIs. Both of these APIs receive a piece of raw data, an <Glossary>access policy</Glossary>, a <Glossary>data transformer</Glossary> and a set of attributes. 

## Create Token API

Create Token stores the raw data in your Token Vault, then creates and returns a token. CreateToken will always generate a unique token for the data. If you want to reuse a token that was already generated, you can use LookupToken. For more details, see the API reference for [Create Token](/reference/post_tokenizer-tokens).

## Lookup Token API

Lookup Token simplifies re-use of existing tokens. It checks the Token Vault for an existing token with the same raw data, <Glossary>data transformer</Glossary> and <Glossary>access policy</Glossary>. If such a token exists, it returns that token. If not, it creates and returns a new token. For more details, see the API reference for [Lookup Token](/reference/post_tokenizer-tokens-actions-lookup).

## Attributes

At token creation, you also have the option to specify additional attributes for that token. These allow for bulk operations on tokens, such as data deletion. _Docs on attributes coming soon! If you need help here, please reach out to [info@userclouds.com](mailto:info@userclouds.com)._
